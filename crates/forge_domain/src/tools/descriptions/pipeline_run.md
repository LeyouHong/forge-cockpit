Runs one of the user's saved pipelines (a DAG of AI-agent and shell nodes) to completion and returns every node's status, outputs, and errors. Prefer this over doing the equivalent work manually whenever an existing pipeline automates the requested task — e.g. run the pr-review pipeline when asked to review a PR. Call pipeline_list first to get the exact file name and the declared inputs, and pass every required input via `inputs`. Nodes run against `dir` (defaults to the current working directory), so set it when the task targets another repo. This call blocks until the whole pipeline finishes; agent nodes may take minutes, so summarize the returned node outputs for the user afterwards instead of re-doing the work.
